Comandos Básicos do Docker

Baixar Imagens Docker

Se você quiser baixar uma imagem sem criar um container, use o comando docker pull.

Por exemplo, para baixar a imagem do Ubuntu:

docker pull ubuntu

Para especificar uma versão, você pode adicionar a tag correspondente:

docker pull ubuntu:3.0

ou caso queira pegar a ultima versão

docker pull ubuntu:latest

Criar e Executar um container

O comando principal para criar e executar containeres é o docker run.

Para criar um container a partir da imagem do Ubuntu, use:

docker run --name meuConteiner ubuntu

Aqui, estamos criando um container chamado "meuConteiner" a partir da imagem do Ubuntu. Se você não tem essa imagem baixada, o Docker irá baixá-la automaticamente antes de criar o container.

Verificar containeres em Execução

Para verificar se o container foi criado e está rodando, use:

docker ps

Este comando mostra todos os containeres que estão rodando. Para ver todos os containeres, incluindo os que estão parados, use:

docker ps -a

Acessar o Terminal do container

Para acessar o terminal do container "meuConteiner", use o comando docker exec com as flags -it e bash:

docker exec -it meuConteiner bash

Agora você está no terminal do seu container e pode interagir com ele como se estivesse em um sistema Ubuntu normal.

Parar um container

Para parar um container em execução, use:

docker stop meuConteiner

Depois de parar, se você digitar docker ps, verá que o "meuConteiner" não está mais listado, pois este comando mostra apenas os containeres em execução.

Excluindo containeres e Imagens

Depois de aprender como criar e parar containeres, vamos ver como removê-los quando não precisamos mais deles.

Removendo containeres

Vamos supor que você terminou de usar o container "meuConteiner" e quer removê-lo para liberar espaço. Para isso, usamos o comando docker rm.

Primeiro, precisamos garantir que o container está parado:

docker stop meuConteiner

Agora, podemos removê-lo com:

docker rm meuConteiner

Se você quiser remover todos os containeres parados de uma vez, use o comando:

docker container prune

Este comando irá perguntar se você realmente quer remover todos os containeres parados. Digite "y" e pressione Enter para confirmar.

Removendo Imagens

Imagens Docker ocupam espaço no disco, então é importante remover aquelas que não são mais necessárias. Para remover uma imagem, usamos o comando docker rmi.

Primeiro, liste todas as imagens para encontrar o ID ou o nome da imagem que deseja remover:

docker images

Agora, remova a imagem usando o ID ou o nome da imagem. Por exemplo:

docker rmi ubuntu:latest

Se você quiser remover todas as imagens que não estão sendo usadas por containeres ativos, use:

docker image prune

E para uma limpeza mais agressiva, que remove todas as imagens, containeres parados, volumes não usados e redes, use:

docker system prune

Criando containeres com Portas

Vamos supor que você quer rodar um servidor web Nginx e precisa expor a porta 80 do container para que seja acessível externamente. Você pode fazer isso com o comando docker run e a flag -p.

Por exemplo:

docker run -d -p 80:80 --name nginx-docker nginx

Aqui, o -d significa "detached mode", que executa o container em segundo plano. A flag -p 80:80 mapeia a porta 80 do host para a porta 80 do container, permitindo que você acesse o servidor Nginx no seu navegador através do endereço http://localhost:80.

Visualizando Logs do container

Se você quiser ver os logs gerados por um container para depuração ou monitoramento, use o comando docker logs.

Por exemplo:

docker logs meuConteiner

Isso mostra todos os logs gerados pelo container "meuConteiner".

Para visualizar os logs em tempo real, use a flag -f:

docker logs -f meuConteiner

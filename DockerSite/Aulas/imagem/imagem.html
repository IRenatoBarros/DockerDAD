Conceito de imagem

Conceito de imagem

Uma imagem Docker é uma representação estática de um aplicativo ou serviço, incluindo sua configuração e dependências, encapsulada em um arquivo executável independente. Essa imagem é usada como modelo para criar containeres, que são instâncias em execução dessa imagem no host do Docker. As imagens são criadas a partir de código de software, dependências, bibliotecas e um Dockerfile, que contém instruções para construir a imagem. Elas são compostas por camadas somente leitura e são imutáveis, o que significa que não podem ser modificadas após a criação. Se for necessário fazer alterações, uma nova imagem deve ser criada. Por outro lado, os containeres são mutáveis e permitem modificações durante o runtime, como a instalação de software ou a modificação de configurações.

Exemplo

Pense em uma imagem Docker como uma planta baixa de uma casa. A planta baixa contém todas as informações necessárias para construir a casa, como o layout dos cômodos, as instalações elétricas e hidráulicas, e os materiais de construção necessários. Da mesma forma, uma imagem Docker contém todas as instruções e componentes necessários para criar um container e executar a aplicação.
Agora, imagine que você tem várias casas idênticas para construir em diferentes locais. Em vez de desenhar uma nova planta baixa para cada casa, você pode fazer uma cópia da planta baixa original e usá-la como modelo para construir as outras casas. Da mesma forma, uma imagem Docker pode ser compartilhada e reutilizada para criar vários containeres executando a mesma aplicação em diferentes ambientes.
Assim como a planta baixa é estática e não pode ser alterada após ser impressa, uma imagem Docker é imutável e não pode ser modificada após sua criação. Se você precisar fazer alterações na planta baixa, precisará desenhar uma nova versão. Da mesma forma, se você precisar fazer alterações na imagem Docker, precisará criar uma nova imagem com as modificações desejadas.
